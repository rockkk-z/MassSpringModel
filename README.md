# 质点弹簧模型——布料物理模拟与数值积分方法对比

## 一、实验目标

1. **掌握动态场景渲染**：了解并使用 Taichi 框架构建 3D 场景，学习使用 Taichi GGUI 编写交互面板。
2. **理解质点-弹簧模型**：掌握基于物理的弹力与阻尼力计算方法，并处理数值爆炸问题（如速度钳制）。
3. **对比数值积分方法**：独立编写并比较三种常见的数值积分求解器（显式欧拉、半隐式欧拉、隐式欧拉），观察并理解它们在物理模拟中的稳定性差异。
4. **理解 GPU 编程基础**：学习 Taichi 中的 `ti.kernel` 与 `ti.func`，了解并行计算中的状态同步与 Kernel 启动开销优化。
5. **选做内容**：完善弹簧模型（增加剪切弹簧、弯曲弹簧），实现质点与球体的空间碰撞处理。

## 二、实验原理

### 2.1 质点-弹簧模型

将布料离散为 $$N \times N$$ 的质点网格，质点间通过弹簧相连。根据胡克定律，两个质点 $a$ 和 $b$ 之间的弹力为：

$$
f_{a} = -k_s (|x_a - x_b| - l) \frac{x_a - x_b}{|x_a - x_b|}
$$

其中 $k_s$ 为劲度系数，$l$ 为弹簧原长，$x$ 为质点位置。

为抑制无限振动并保证系统稳定，引入阻尼力：

$$
f_d = -k_d v_a
$$

每个质点还受到重力 $$f_g = mg$$，总受力为弹力、阻尼力、重力的累加。

### 2.2 数值积分方法

基于牛顿第二定律 $$a = F/m$$，在离散时间步 $\Delta t$ 内更新速度与位置。

- **显式欧拉**：完全使用当前时刻状态计算下一时刻。
  $$
  v_{t+1} = v_t + a_t \Delta t, \quad x_{t+1} = x_t + v_t \Delta t
  $$
- **半隐式欧拉**：先更新速度，再用新速度更新位置。
  $$
  v_{t+1} = v_t + a_t \Delta t, \quad x_{t+1} = x_t + v_{t+1} \Delta t
  $$
- **隐式欧拉**：使用未来状态计算加速度，形成自洽方程。
  $$
  v_{t+1} = v_t + a(x_{t+1}, v_{t+1}) \Delta t, \quad x_{t+1} = x_t + v_{t+1} \Delta t
  $$
  本实验采用定点迭代法（Fixed-Point Iteration）近似求解：将当前状态作为初值，多次迭代更新 $x_{next}, v_{next}$ 直至收敛。

## 三、实验任务与实现

### 3.1 场景初始化与 GPU 同步

初始化拆分为多个 `@ti.kernel`，在 Python 侧顺序调用，保证多线程 GPU 计算的状态同步。

- `init_positions()`：初始化质点位置、速度、受力，固定左上与右上角点。
- `init_springs()`：构建弹簧拓扑，包括：
  - **结构弹簧**：水平/垂直相邻质点
  - **剪切弹簧**：两条对角线方向
  - **弯曲弹簧**：间隔一行/列的结构质点
- `init_spring_indices()`：将弹簧端点压扁为连续索引，供渲染使用。

每次重置时弹簧计数器清零，避免重复调用时累加。

### 3.2 力学计算与防爆处理

`compute_forces_on()` 为 `@ti.func`，被内联到调用它的 kernel 中。计算分两个阶段：
1. 清空受力，并行施加重力与阻尼力（无冲突）。
2. 遍历所有弹簧，用 `ti.atomic_add` 累加弹簧力，避免多线程写入冲突。

`clamp_velocity()` 限制质点最大速度，防止显式方法中数值爆炸。

### 3.3 积分求解器实现

遵循实验要求，每个积分器都在**单个 kernel** 内完成全部计算，最小化 GPU 启动开销。

- `step_explicit()`：顺序更新位置（旧速度）→ 速度（旧力）→ 钳制 → 碰撞解决。
- `step_semi_implicit()`：先更新速度（旧力）→ 钳制 → 新速度更新位置 → 碰撞解决。
- `step_implicit_iter()`：使用 `ti.static` 编译期展开 3 次定点迭代。每次迭代用当前预测状态计算力，更新预测速度、位置并进行碰撞解决，最终写回真实场。

### 3.4 渲染与 GGUI 交互

- 使用 `ti.ui.Window` 创建 800×800 窗口。
- GUI 面板包含：
  - 三种积分方法的单选按钮（带 `[*]` 选中标识）。
  - 暂停/恢复按钮。
  - 重置布料按钮。
- 切换方法时自动重置布料，避免不稳定的状态传递。
- 相机支持鼠标右键拖拽旋转视角。

### 3.5 选做内容实现

#### 增强弹簧模型
在 `init_springs()` 中添加了剪切弹簧与弯曲弹簧：
- 剪切弹簧：连接 `(i,j)` 与 `(i+1,j+1)`、`(i,j)` 与 `(i-1,j+1)`，抵抗面内剪切。
- 弯曲弹簧：连接 `(i,j)` 与 `(i+2,j)`、`(i,j)` 与 `(i,j+2)`，抵抗弯曲变形。

这些弹簧使布料形态更接近真实织物，抗弯、抗剪能力显著增强。

#### 球体碰撞处理
在场景 `(0, 0.3, 0)` 处放置半径 0.15 的球体。`resolve_sphere_collision()` 函数：
- 检测质点是否进入球体内部。
- 若穿透，将质点沿径向推出到球体表面。
- 若质点速度有指向球心的分量，移除该法向分量，实现无弹性滑动。

碰撞检测在每个积分步的位置更新后立即执行，隐式欧拉中还在每次定点迭代内进行，确保预测状态也不穿透。

### 4.3 数值积分方法稳定性对比

在相同参数（`k_s=10000`, `dt=5e-4`）下，三种方法表现如下：

| 方法       | 稳定性 | 振荡情况 | 实时性 |
|------------|--------|----------|--------|
| 显式欧拉   | 极差   | 迅速爆炸 | 正常   |
| 半隐式欧拉 | 良好   | 轻微振荡 | 正常   |
| 隐式欧拉   | 极佳   | 基本无振荡 | 稍慢   |

显式欧拉即使在较大阻尼下也可能逐步发散，符合理论预期——显式法对于刚性系统条件数敏感。半隐式欧拉能量守恒较好，成为本实验默认方法。隐式欧拉无条件稳定，但 3 次定点迭代引入额外数值阻尼，表现为布料下落后快速静止。

### 4.4 阻尼系数的影响

以半隐式欧拉为例，增大阻尼系数 `k_d`：
- `k_d = 1.0`：布料下落过程中有明显回弹，振荡次数较多。
- `k_d = 5.0`：振荡被迅速抑制，布料更快趋于平衡位置。

阻尼力直接消耗系统动能，合理设置阻尼能平衡实时性与视觉稳定性。

### 4.5 增强弹簧模型效果

添加剪切和弯曲弹簧后，布料在静止状态下更平整，悬挂时弯曲角度更自然，抵抗局部变形能力增强。结构弹簧仅能维持网格边长，而剪切弹簧防止菱形化，弯曲弹簧使布料具有一定的“挺括”感。

### 4.6 球体碰撞效果

布料下落至球体时能自然包裹、滑动，质点无穿透现象。碰撞响应简单高效，验证了位置校正与速度修正方法的可行性。

通过本次实验：

- 深入理解了质点-弹簧模型与三种数值积分方法的原理及稳定性差异。
- 实践了 `@ti.kernel` 与 `@ti.func` 的合理划分，以最小化 GPU 启动开销。
